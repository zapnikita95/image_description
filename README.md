# Image Description

Инструмент для тех, кто **живёт в фидах и карточках**: берёте YML/XML с офферами, по URL картинок прогоняете **локальную vision-модель в Ollama** (линейка **Qwen 3.5** и совместимые), на выходе — **структурированные атрибуты** под ваши направления (в т.ч. текст на товаре, если включено). Без облачных vision-API, если у вас уже есть GPU и политика «данные не уезжают наружу».

**Кому заходит:** ecom-аналитики и контент, мерчандайзинг, PIM и интеграторы выгрузок на маркетплейсы — когда нужно **массово добить карточку тем, что реально видно на фото**, а не размечать тысячи SKU вручную.

**Что умеет:** проекты с вертикалями и кастомными атрибутами, потоковый парсинг крупных фидов, пауза/продолжение очереди, переобработка строк с низкой уверенностью, **Gradio-интерфейс** и **CLI** (`run.py`) для скриптов и CI, выгрузка JSON/CSV. Опционально — **дообучение** на своих данных (в т.ч. DeepFashion) и сравнение качества до/после.

**Очередь на несколько клиентов:** `run.py` по умолчанию ходит на `http://127.0.0.1:11435` — типичный порт **ollama-queue-proxy** перед чистым Ollama на `:11434`. В приложении URL задаётся в настройках; прямой Ollama тоже поддерживается.

## Требования

- **Python** 3.9 или новее (в PATH: команда `python3` на Mac/Linux, `python` на Windows).
- **Ollama** 0.12.7 или новее, установлен и запущен (по умолчанию слушает `http://localhost:11434`).
- **Vision-модель** в Ollama — линейка **Qwen 3.5** (не 2.5). В интерфейсе (Настройки) выбор: **4B**, **9B** или **35B MoE**. По умолчанию — **qwen3.5:35b** (sweet spot для 24 ГБ, макс. качество по картинкам). Опционально — **qwen3.5:35b-a3b** (или своё имя), если импортировали A3B GGUF: картинки обрабатывает та модель, которая выбрана в Настройках или в блоке «Оценка до/после». Для извлечения атрибутов — именно qwen3.5 (или своя на его базе); qwen2.5-vl нужен для дообучения, не для инференса по умолчанию.

**Дообучение: 2.5-VL и Qwen3-VL.** В интерфейсе (вкладка «Дообучение» → базовая VL-модель) можно выбрать:
- **Qwen2.5-VL** (3B, 7B, 72B) — меньше VRAM, проверенный пайплайн.
- **Qwen3-VL** (4B, 8B, 32B) — новое поколение vision-моделей, сильнее по картинкам; 32B даёт макс. качество при дообучении (нужна видеокарта с запасом VRAM).

После обучения и экспорта в Ollama дообученная модель появляется в списке моделей в Настройках — её можно использовать для распознавания наравне с qwen3.5:35b. Для инференса по умолчанию по-прежнему удобно брать **qwen3.5:35b**; если нужны свои знания по одежде у «мощной» модели — дообучи **Qwen3-VL-32B-Instruct** и экспортируй в Ollama.

## Установка по шагам

1. Установите Ollama с официального сайта: https://ollama.com
2. В терминале выполните (один раз) для загрузки vision-модели:
   - **24 ГБ ГПУ (RTX 3090 и т.п.):** `ollama pull qwen3.5:35b` — модель по умолчанию в приложении, максимум качества по картинкам (MoE, ~24 ГБ).
   - Меньше VRAM: `ollama pull qwen3.5:9b` или `ollama pull qwen3.5:4b`. В Настройках выберите нужный вариант (4B / 9B / 35B MoE).
   - **Опционально (GGUF, лучший баланс качество/скорость/VRAM):** скачайте GGUF (например `unsloth/Qwen3.5-35B-A3B-GGUF` или `bartowski/Qwen_Qwen3.5-35B-A3B-GGUF`) — квантизация **Q4_K_XL** или **Q5_K_M**. Затем создайте модель в Ollama: положите GGUF в папку и выполните `ollama create <имя> -f Modelfile` (в Modelfile строка `FROM ./файл.gguf`). В Настройках приложения укажите это имя модели.
3. Перейдите в каталог проекта (туда, где лежат `run.py` и `requirements.txt`).
4. Создайте виртуальное окружение и установите зависимости (опционально, но рекомендуется):
   - Mac/Linux:
     ```bash
     python3 -m venv venv
     source venv/bin/activate
     pip install -r requirements.txt
     ```
   - Windows (CMD):
     ```cmd
     python -m venv venv
     venv\Scripts\activate
     pip install -r requirements.txt
     ```

## Запуск

**Одежда — только применимые атрибуты:** в `config.json` проекта есть флаг `dynamic_clothing_attributes` (по умолчанию `true`). Включение/выключение — на вкладке **«Настройки»**, блок проекта, **только если вертикаль «Одежда»** (галка не показывается для других вертикалей). Сохраните настройки кнопкой **«Сохранить всё»**. Тогда модель возвращает JSON **только с теми ключами**, которые логичны для типа товара (без «длины рукава» у трусов и т.п.).

**`text_enabled` в JSON направлений** — флаг направления: **искать на фото надписи и принты на товаре**. Отдельно в **Настройках** проекта галка **«Искать надписи на товаре»** (`extract_inscriptions`): если её **выключить**, отдельный запрос про текст **не выполняется** (даже при `text_enabled` в JSON). Базовые атрибуты одежды задаются в **`attributes`**. На карточке «Результаты» блок про надписи — только если включены и галка, и `text_enabled`. На **Запуск** можно задать **несколько** «Извлекаемых атрибутов» (строки и кнопки +/−): в промпт попадут все; по ключам в скобках список атрибутов в запросе к модели **сужается** до указанных полей.

**Пауза и продолжение:** после **⏹ Остановить** очередь сохраняется в `projects/<проект>/pending_run.json`. Следующий **▶ Запустить** с **теми же** лимитом, категориями и режимом перезаписи продолжит **остальные** офферы, а не новую выборку с нуля. Изменили лимит/категории — начнётся новый подбор (старый pending сбрасывается).

**Повтор с вкладки «Результаты»:** свёрнутый блок **«Повторная обработка низкой уверенности»** — по текущему фильтру (ползунки и категория) можно задать порог, кнопкой **«Выбрать все ниже порога»** или вручную в мультивыборе отметить офферы, выбрать **другую модель** Ollama и перезаписать строки в БД; ход работы смотрите в терминале (`[Reprocess]`).

**Ювелирка / формулировки в ответе модели:** **вертикаль проекта** в «Настройках» должна соответствовать сфере (для украшений — не оставляйте **«Одежда»**, иначе в промпт и направления подтянутся шаблоны одежды). Универсальный текст в приложении без отраслевых примеров; нюансы (как называть оттенок металла и т.д.) задавайте в **«Доп. ограничения»** и **«Примеры»** на вкладке «Запуск». Ключ в JSON — как в настройках направления (`key`), синонимы из ответа модели подставляются к нему.

**Лимит офферов без категорий:** число N означает «взять **N ещё не обработанных** офферов». Если первые записи в кэше уже есть в результатах, подбор **продолжается дальше по фиду**, пока не наберётся N новых (а не обнуляется после первых совпадений).

**Скрипты проверки модели** (нужны Ollama и скачанная модель, например `qwen3.5:9b`):
- `python scripts/run_jewelry_color_test.py` — цвет украшения по фото;
- `python scripts/run_metal_color_test.py` — цвет **металла** (не камней), результат в `generated/metal_color_test/last_run.txt`.
- `python scripts/run_block_prompt_jewelry_test.py` — короткая задача + поля «ограничения» и «примеры» в проекте, проверка сборки промпта.

**Промпт для не-одежды:** к полю «Задание» автоматически добавляются блоки: вертикаль, фокус на товаре, **источник истины — фото** (если название противоречит картинке, верить глазам), системные ограничения, **ваши** доп. ограничения и примеры (если заполнены), формат JSON / русский / уверенность. Заполняйте задание кратко; детали — в «Ограничения» и «Примеры» **только на вкладке «Запуск»** (не в `config.json` проекта).

Поле **«Извлекаемый атрибут»** — **только «Запуск»**: кратко укажите, что измеряете (например `Цвет металла (metal_color)`). Текст попадёт в промпт блоком «ИЗВЛЕКАЕМЫЙ АТРИБУТ»; ключ в JSON задаётся в направлениях проекта (поле `key` у атрибута). Последние введённые поля запуска сохраняются в **`run_prompt_last.json`**. **Именованные пресеты** (несколько промптов на выбор) хранятся **отдельно в каждом проекте**: `projects/<имя проекта>/prompt_presets.json` — вкладка «Запуск»: выбор пресета в списке (поля промпта скрываются), **«Изменить пресет»** / **«Новый пресет»**, **«Сохранить как пресет»** (в пресет входят поля задания и окно «итоговый промпт»).

**Обычный путь:** заполняете поля «Задание», ограничения, примеры, атрибут — при запуске полный запрос к модели **собирается автоматически** (смотреть его не нужно). **Дополнительно** (свёрнутый блок на «Запуск»): предпросмотр или полная ручная подмена текста для экспертов; плейсхолдер `{product_name}`.

**Вкладка «Фид»:** можно ввести **полный путь** к XML/YML или **перетащить файл** в область загрузки. Для **очень больших** фидов (сотни МБ и больше) нужен пакет **`lxml`** (уже в `requirements.txt`): парсинг идёт **потоково**, без загрузки всего файла в память. «Битые» или нестандартные XML (другая кодировка, мусор в начале) обрабатываются устойчивее, чем стандартным парсером Python.

Из корня проекта выполните одну команду — откроется браузер с приложением:

### Запуск кликом по файлу

- **Mac:** Двойной клик по файлу `run.command`. При первом запуске может потребоваться: ПКМ по файлу → «Открыть» или в «Системных настройках → Безопасность» разрешить запуск. Откроется Терминал, скрипт запросит путь к YML-фиду (если не передан аргументом).
- **Windows:** Двойной клик по файлу `run.bat`. Окно CMD откроется и останется открытым после завершения. Если Python не в PATH — установите Python с галочкой «Add Python to PATH» или укажите полный путь к `python.exe` в первой строке `run.bat`.
- **Linux:** Двойной клик по `run.sh` (если файловый менеджер предлагает «Run in Terminal») или из терминала: `./run.sh`. При необходимости выполните: `chmod +x run.sh`.

По умолчанию лаунчеры запускают обработку с `--limit 5` (первые 5 офферов с картинками). Путь к фиду и выходному файлу можно ввести при запросе или передать аргументами при запуске из терминала.

### Запуск из терминала

Команда (обязательные аргументы: путь к фиду; остальное опционально):

```bash
python3 run.py --feed /path/to/feed.yml --output results.json --limit 10
```

- `--feed`, `-f` — путь к YML-фиду (обязательный, если не введён интерактивно).
- `--output`, `-o` — путь к выходному файлу (по умолчанию: `feed_image_attributes_results.json`).
- `--limit`, `-n` — максимум офферов для обработки (0 = все). По умолчанию при запуске через лаунчер используется 5.
- `--ollama-url` — базовый URL Ollama (по умолчанию: `http://localhost:11434`).
- `--model`, `-m` — имя модели Ollama (по умолчанию: `qwen3.5:2b`).
- `--format` — формат вывода: `json` или `csv`. Можно не указывать, если расширение выходного файла `.csv`.
- `--no-interactive` — не запрашивать путь к фиду и выходу, если они не заданы (для скриптов).

Примеры:

- Mac/Linux:
  ```bash
  cd /path/to/feed_image_attributes
  source venv/bin/activate   # если используете venv
  python3 run.py --feed ~/Downloads/feed.xml --output out.json --limit 20
  ```
- Windows (CMD):
  ```cmd
  cd C:\path\to\feed_image_attributes
  venv\Scripts\activate
  python run.py --feed C:\feeds\feed.xml --output out.csv --limit 20
  ```

### Профилирование скорости анализа (Gradio)

Переменная окружения **`IMAGE_DESC_PROFILE=1`** (или `true`): в результате анализа оффера добавляется **`_profile`** (время подготовки картинки, суммарное время vision-вызовов, список вызовов по задачам); краткая сводка попадает в лог вкладки «Запуск» и в stderr. В **Настройках** — **«Параллельных vision-запросов к Ollama»**: **`0`** — без лимита (как раньше), **`1`–`2`** — часто удобнее на одной GPU.

Одна картинка из терминала: `python scripts/profile_one_image.py --profile <url_или_путь>`.

Замер без Gradio на вашей машине (локальный JPEG + Ollama на **127.0.0.1**): `python test_profile_benchmark.py` — пишет `benchmark_results.txt` и `benchmark_results.json`.

## Входные данные

Ожидается YML-фид (XML) с офферами. В каждом оффере должен быть хотя бы один URL картинки в одном из тегов: `<picture>`, `<image>`, `<picture_url>`. Используется первая найденная картинка по офферу.

Пример фрагмента:

```xml
<offer id="123">
  <name>Футболка с принтом</name>
  <picture>https://example.com/photo/123.jpg</picture>
</offer>
```

Поддерживаются фиды с namespace (например `http://www.shopcreator.ru/shop/yml`).

---

## Выходные данные

Формат зависит от выбора: JSON (по умолчанию) или CSV.

Поля:

- `offer_id` — идентификатор оффера из фида.
- `name` — название товара из фида.
- `picture_url` — URL картинки, по которой выполнялся запрос.
- `text_on_clothing` — прочитанный текст на одежде (или «нет», если текста нет).
- `description_adjectives` — описание прилагательными через запятую.

При ошибке загрузки картинки или вызова модели в JSON добавляется поле `error`; в CSV соответствующие ячейки остаются пустыми.

Пример одной строки вывода (JSON):

```json
{
  "offer_id": "123",
  "name": "Футболка с принтом",
  "picture_url": "https://example.com/photo/123.jpg",
  "text_on_clothing": "Hello World",
  "description_adjectives": " casual, белая, с принтом, хлопок, короткий рукав"
}
```

## Частые проблемы (для LLM)

- **«Ollama не запущен» / Connection refused**  
  Проверить: в браузере или `curl http://localhost:11434` — сервис должен отвечать. Запустить Ollama с рабочего стола или из терминала: `ollama serve` (или просто открыть приложение Ollama).

- **«Нет картинок в фиде» / обрабатывается 0 офферов**  
  В YML в офферах должны быть теги `<picture>`, `<image>` или `<picture_url>` с URL (http/https). Проверить один оффер в фиде на наличие такого тега.

- **«Модель не найдена» / model not found**  
  Выполнить: `ollama pull qwen3.5:2b` (или тот тег, который указан в `--model`). Убедиться, что тег совпадает с именем в `--model`.

- **«Python не найден» / python: command not found**  
  Установить Python 3.9+ и добавить его в PATH. На Windows при установке Python включить опцию «Add Python to PATH». В лаунчерах при необходимости заменить `python3` / `python` на полный путь к исполняемому файлу.

- **Окно закрывается сразу после запуска**  
  Лаунчеры уже содержат `pause` (Windows) или `read -p "Нажмите Enter..."` (Mac/Linux). Если окно всё равно закрывается, запускать из терминала: `python3 run.py --feed /path/to/feed.yml` и смотреть вывод ошибок.

## Модель на английском, интерфейс на русском (2026)

Атрибуты в промпте и в ответе модели задаются на **английском** (short, long, sleeveless, crew, v-neck, plain, striped и т.д.). В интерфейсе приложения значения переводятся на русский через глоссарий `attribute_glossary.json` в корне проекта. В БД и в логах сырых ответов хранятся английские значения.

## Датасет DeepFashion и дообучение (2026)

Бенчмарк **DeepFashion Category and Attribute Prediction** — стандартный набор для атрибутов одежды (~289k изображений, аннотации на английском).

- Официальная страница: [DeepFashion Attribute Prediction](https://mmlab.ie.cuhk.edu.hk/projects/DeepFashion/AttributePrediction.html)
- Скачивание: на странице указаны ссылки (Google Drive / Dropbox / Baidu). Нужны: изображения («Clothes Images» или High-Res) и аннотации: `list_attr_cloth.txt`, `list_attr_img.txt` (в каталоге `Anno/`).

### Как запустить (пошагово)

Все команды — из **корня проекта** (там, где лежат `app.py`, `scripts/`). На Windows используйте `python` (или `.\venv\Scripts\python.exe`, если включён venv).

| Шаг | Что делаем | Команда |
|-----|------------|--------|
| 0 | Установить gdown (для скачивания с Google Drive) | `pip install gdown` |
| 1 | Скачать датасет | `python scripts/download_deepfashion.py --out data/deepfashion` |
| 2 | Конвертировать в JSONL (первые 500 примеров) | `python scripts/convert_deepfashion_to_jsonl.py --root data/deepfashion --out data/deepfashion/train.jsonl --max-examples 500` |
| 3 | Запустить приложение | `python app.py` → в браузере вкладка «Дообучение» |
| 4 | В приложении: указать «Внешний датасет (JSONL)» = `data/deepfashion/train.jsonl`, нажать «Собрать датасет», затем запустить обучение и экспорт в Ollama | — |
| 5 | Оценка «до» (сохранить ответы базовой модели) | `python scripts/eval_before_after.py --images data/deepfashion/train.jsonl --out results --phase before` |
| 6 | После дообучения: переключить в настройках модель на новую (например `clothes-detector-v1`) | — |
| 7 | Оценка «после» | `python scripts/eval_before_after.py --images data/deepfashion/train.jsonl --out results --phase after` |
| 8 | Сравнение до/после | `python scripts/eval_before_after.py --out results --phase compare` |

Результат сравнения: в консоли и в `results/eval_report.txt`. Для eval нужна запущенная Ollama и подтянутая модель (например `ollama pull qwen3.5:35b`).

Если скрипт не сработал: открой [Google Drive](https://drive.google.com/drive/folders/0B7EVK8r0v71pQ2FuZ0k0QnhBQnc) → **Category and Attribute Prediction Benchmark**. Дальше: папка **Img** → скачай **img.zip** (2.5 ГБ), распакуй так, чтобы картинки оказались в `data/deepfashion/img/`. Папка **Anno_coarse** → скачай **list_attr_cloth.txt** и **list_attr_img.txt**, положи их в `data/deepfashion/Anno/`. Итог: в `data/deepfashion` есть `Anno/` (два файла) и `img/` (картинки). Папки Anno_fine, Eval и img_highres.zip не нужны.

### Скачивание и конвертация DeepFashion (подробно)

1. **Скачать датасет** в каталог `data/deepfashion` (или указать `--out`):
   ```bash
   python scripts/download_deepfashion.py --out data/deepfashion
   ```
   Если авто-загрузка не сработала: в [Google Drive](https://drive.google.com/drive/folders/0B7EVK8r0v71pQ2FuZ0k0QnhBQnc) зайдите в **Category and Attribute Prediction Benchmark** → папка **Img** → скачайте **img.zip** (2.5 ГБ); папка **Anno_coarse** → скачайте **list_attr_cloth.txt** и **list_attr_img.txt**. Распакуйте img.zip в `data/deepfashion/img/`, два txt положите в `data/deepfashion/Anno/`.

2. **Конвертировать в JSONL** (пути к картинкам и словарь атрибутов). Параметр **`--out`** (не `--output`):
   ```bash
   python scripts/convert_deepfashion_to_jsonl.py --root data/deepfashion --out data/deepfashion/train.jsonl --max-examples 500
   ```
   Для быстрых прогонов используйте `--max-examples`. Без него конвертируются все строки из `list_attr_img.txt`.

3. **Дообучение в приложении:** вкладка «Дообучение» — укажите путь к внешнему датасету «Внешний датасет (JSONL)» (например `data/deepfashion/train.jsonl`). Соберите датасет (правки + внешний JSONL объединяются в один `train.jsonl`), запустите обучение и экспорт в Ollama. В настройках выберите новую модель (например `clothes-detector-v1`).

### Оценка до/после дообучения

Скрипт `scripts/eval_before_after.py` запускает модель на одном и том же наборе картинок «до» и «после» дообучения и выводит сравнение атрибутов.

1. **Фаза «до»** — сохранить ответы текущей (базовой или уже дообученной) модели:
   ```bash
   python scripts/eval_before_after.py --images data/deepfashion/train.jsonl --out results --phase before
   ```
   Вместо JSONL можно указать каталог с изображениями: `--images path/to/folder`.

2. Провести дообучение и экспорт в Ollama, переключить в настройках приложения модель на новую.

3. **Фаза «после»** — те же картинки, новая модель:
   ```bash
   python scripts/eval_before_after.py --images data/deepfashion/train.jsonl --out results --phase after
   ```

4. **Сравнение** — совпадения и расхождения атрибутов:
   ```bash
   python scripts/eval_before_after.py --out results --phase compare
   ```
   Результат: в консоли и в `results/eval_report.txt` — количество совпадений и примеры изменений.

Опции: `--model` — имя модели Ollama (по умолчанию из настроек приложения); `--ollama-url` — URL Ollama.

### Лексика DeepFashion: извлечение ключей и перевод EN→RU

Чтобы использовать в конфиге направлений полный словарь атрибутов из датасета (и отображать их по-русски в UI), можно собрать ключи из JSONL, перевести их на RU через Ollama и применить direction'ы к проекту.

1. **Извлечь уникальные EN-ключи** из ShareGPT-формата JSONL:
   ```bash
   python scripts/extract_attribute_keys_from_sharegpt_jsonl.py --jsonl fine_tune/dataset/train_mini_vl.jsonl --out generated/deepfashion/attribute_keys.json
   ```

2. **Перевести ключи в RU-лейблы** (Ollama, с кэшем и ретраями при таймаутах):
   ```bash
   python scripts/translate_deepfashion_attribute_labels.py --keys generated/deepfashion/attribute_keys.json --out-cache generated/deepfashion/attribute_labels_ru.json --glossary attribute_glossary.json --ollama-url http://localhost:11434 --translate-model qwen3.5:4b --batch-size 10 --timeout 180
   ```
   Прогресс сохраняется после каждого успешного батча; при падении можно перезапустить — скрипт продолжит с недостающих ключей.

3. **Собрать direction'ы чанками** (по 80 ключей на направление):
   ```bash
   python scripts/build_deepfashion_directions_from_labels.py --keys generated/deepfashion/attribute_keys.json --labels generated/deepfashion/attribute_labels_ru.json --out generated/deepfashion/deepfashion_directions.json --keys-per-direction 80 --direction-prefix deepfashion_chunk
   ```

4. **Применить direction'ы к проекту** (добавить в `projects/<name>/config.json`):
   ```bash
   python scripts/apply_deepfashion_directions_to_project.py --project Befree --directions generated/deepfashion/deepfashion_directions.json
   ```
   Опция `--update-existing` обновляет лейблы у уже добавленных direction'ов (удобно после дозаполнения перевода).

Артефакты лежат в `generated/deepfashion/` и переиспользуются для любых проектов.

## Перенос на другой компьютер

1. Соберите архив проекта (без виртуального окружения и кэша). Из каталога проекта выполните:
   - **Mac:** двойной клик по `make_archive.command` или в терминале: `./make_archive.sh`
   - **Linux:** в терминале: `./make_archive.sh`
   - **Windows:** двойной клик по `make_archive.bat`
   Или вручную (Mac/Linux):  
   `zip -r feed_image_attributes.zip . -x "venv/*" "__pycache__/*" "*.pyc" ".env" "*.zip"`  
   На Windows: двойной клик по `make_archive.bat` или упаковать папку в zip вручную, исключив каталоги `venv` и `__pycache__`.
2. Перенесите файл `feed_image_attributes.zip` на другой компьютер и распакуйте в любую папку.
3. На новом компьютере: установить Ollama, подтянуть модель (`ollama pull qwen3.5:2b`), установить Python и зависимости (`pip install -r requirements.txt`), затем запускать по README (кликом по `run.command` / `run.bat` / `run.sh` или из терминала с `--feed` и `--output`).
